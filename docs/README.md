# Documentación del proyecto

Esta carpeta concentra el contexto persistente del repo para humanos y agentes.

## Mapa

- `architecture.md`: arquitectura y límites entre sistemas
- `domain/`: reglas de negocio y semántica del producto
- `integrations/`: integraciones externas y contratos entre sistemas
- `runbooks/`: guías operativas
- `changes/`: especificaciones de cambios activos o aprobados
- `decisions/`: decisiones duraderas con fecha
- `handoffs/`: notas temporales de trabajo pendiente

## Uso recomendado

1. Lee `README.md` del repo.
2. Lee `CLAUDE.md`.
3. Lee `AGENTS.md`.
4. Entra luego a la doc de dominio o integración que corresponda.
